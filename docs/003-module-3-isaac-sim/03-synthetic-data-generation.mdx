# Synthetic Data Generation in Isaac Sim

## Introduction

Synthetic data generation is a powerful capability of Isaac Sim that enables the creation of large, diverse datasets for training AI models. By leveraging Isaac Sim's photorealistic rendering and sensor simulation, you can generate labeled data at scale without the need for physical data collection. In this lesson, you'll learn to set up synthetic data generation pipelines for computer vision and robotics applications.

## Key Concepts in Synthetic Data Generation

### Data Types
Isaac Sim can generate various types of synthetic data:
- **RGB Images**: Photorealistic color images
- **Depth Maps**: Per-pixel depth information
- **Semantic Segmentation**: Pixel-level object classification
- **Instance Segmentation**: Pixel-level object instance identification
- **Bounding Boxes**: 2D/3D object localization
- **Pose Data**: Object position and orientation information

### Sensor Simulation
Realistic sensor models include:
- **RGB Cameras**: With realistic noise, distortion, and exposure
- **Depth Sensors**: Accurate depth information with noise models
- **LiDAR**: 2D and 3D LiDAR simulation with realistic returns
- **IMU**: Inertial measurement units with realistic noise

### Annotation Tools
Built-in annotation capabilities:
- **Automatic Labeling**: Objects are automatically labeled
- **Custom Annotations**: User-defined annotation schemes
- **Multi-modal Data**: Synchronized data from multiple sensors

## Hands-On Lab: Creating a Synthetic Data Pipeline

Let's create a synthetic data generation pipeline that captures RGB images and depth maps from our humanoid robot.

### Synthetic Data Generation Script

```python title="code-examples/isaac-sim-module/synthetic_data_generation.py"
import omni
from omni.isaac.kit import SimulationApp
from omni.isaac.core import World
from omni.isaac.core.utils.stage import add_reference_to_stage
from omni.isaac.core.utils.nucleus import get_assets_root_path
from omni.isaac.core.utils.prims import get_prim_at_path
from omni.isaac.synthetic_utils import SyntheticDataHelper
from pxr import Gf, UsdGeom
import numpy as np
import cv2
import os
from pathlib import Path

# Initialize Isaac Sim application
simulation_app = SimulationApp({"headless": False})

# Get the world instance
world = World(stage_units_in_meters=1.0)

# Add a ground plane
world.scene.add_default_ground_plane()

# Add a simple cube as an object to detect
stage = omni.usd.get_context().get_stage()
cube_path = "/World/Cube"
cube = UsdGeom.Cube.Define(stage, cube_path)
cube.GetSizeAttr().Set(0.2)
cube.GetPrim().GetAttribute("xformOp:translate").Set(Gf.Vec3d(1.0, 0.0, 0.1))

# Add a camera prim
camera_path = "/World/Camera"
from omni.isaac.core.prims import XFormPrim
camera_prim = XFormPrim(prim_path=camera_path, name="camera")

# Create synthetic data helper
synthetic_data = SyntheticDataHelper()

# Create output directory
output_dir = Path("code-examples/isaac-sim-module/synthetic_data_output")
output_dir.mkdir(exist_ok=True)

# Play the simulation
world.play()

# Generate synthetic data
for i in range(50):  # Generate 50 frames
    world.step(render=True)

    # Get RGB image
    rgb_data = synthetic_data.get_rgb_data()
    if rgb_data is not None:
        # Convert to OpenCV format and save
        rgb_image = cv2.cvtColor(rgb_data, cv2.COLOR_RGB2BGR)
        cv2.imwrite(f"{output_dir}/rgb_{i:04d}.png", rgb_image)

    # Get depth data
    depth_data = synthetic_data.get_depth_data()
    if depth_data is not None:
        # Save depth as numpy array
        np.save(f"{output_dir}/depth_{i:04d}.npy", depth_data)

    # Get segmentation data
    seg_data = synthetic_data.get_segmentation_data()
    if seg_data is not None:
        # Convert to image format and save
        seg_image = seg_data.astype(np.uint8)
        cv2.imwrite(f"{output_dir}/seg_{i:04d}.png", seg_image)

    if i % 10 == 0:
        print(f"Generated synthetic data frame {i}")

# Stop the simulation
world.stop()

# Shutdown the application
simulation_app.close()

print(f"Synthetic data saved to {output_dir}")
